# 技术栈决策

日期：2026-04-20

## 背景

MVP 需要确定前端框架、样式方案和状态管理方式。

## 选项

- React + Vite + TypeScript
- 其它前端栈

## 结论

采用 React + Vite + TypeScript + TailwindCSS + shadcn/ui + Jotai。

## 理由

- React 符合当前偏好和项目目标
- Vite 启动快，适合快速迭代
- TypeScript 便于后续扩展字段和状态
- TailwindCSS 和 shadcn/ui 适合小步快跑
- Jotai 比 Context + useReducer 更轻，状态粒度更合适
- 存储层采用 Repository 抽象，先接 localStorage，后续可替换
