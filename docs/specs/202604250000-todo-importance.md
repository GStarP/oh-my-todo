# Todo 重要度分组规格

日期：2026-04-25

## 目标

为 Todo 添加重要度属性，并按重要度分组展示；已完成 todo 单独沉到底部，减少列表占用。

## 数据模型

```ts
interface Todo {
  id: string
  title: string
  completed: boolean
  deadline: string | null
  importance: number
}
```

- `importance` 为整数，默认 `0`
- 无上限
- 旧数据缺失 `importance` 时按 `0` 处理

## 功能描述

### 分组规则

- 未完成 todo 按 `importance` 分组
- 重要度越高的组越在上面
- 已完成 todo 不参与重要度分组，统一进入 `已完成` 特殊组
- `已完成` 组永远排在最底部
- 组内顺序保持创建顺序

### 展开收起

- 所有未完成组默认展开
- `已完成` 组默认收起
- 展开状态不持久化
- 新出现的未完成组默认展开

### 组头展示

- 普通组显示 `重要度 N`
- 已完成组显示 `已完成`
- 左侧显示箭头，随展开状态旋转
- 不显示组内数量

### 编辑入口

- 在 `TodoSidebar` 中新增重要度输入框
- 位置在标题和截止时间之间
- 使用 `type="number"`，`min={0}`
- 修改时实时保存

## 验收标准

- [ ] 新建 todo 默认重要度为 0
- [ ] 侧边栏可编辑重要度，修改后列表实时更新分组
- [ ] 未完成 todo 按重要度分组，重要度高的组在上面
- [ ] 已完成 todo 单独归入 `已完成` 组，并排在底部
- [ ] 所有未完成组默认展开，`已完成` 组默认收起
- [ ] 组头箭头随展开状态变化
- [ ] 组头不显示数量
- [ ] 旧 localStorage 数据正常加载
