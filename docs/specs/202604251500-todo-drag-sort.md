# 拖拽排序规格

日期：2026-04-25

## 目标

支持通过拖拽改变 todo 的组内排序和跨组移动，提供直观的排序交互。

## 数据模型

```ts
interface Todo {
  id: string
  title: string
  completed: boolean
  deadline: string | null
  importance: number
  sortOrder: number
}
```

- `sortOrder` 为整数，默认 `0`
- 旧数据缺失 `sortOrder` 时按 `0` 处理
- 组内排序：同 `importance` 的 todo 按 `sortOrder` 升序排列
- 拖拽完成后批量重算受影响 todo 的 `sortOrder`，避免间隙

## 功能描述

### 触发

- 长按 todo 500ms 进入拖拽状态
- 触摸和鼠标均支持
- 长按期间轻微缩放反馈，表示正在准备拖拽

### 拖拽中

- 被拖拽 todo 跟随指针移动，带阴影
- 原位置留空（占位符），高度与原 todo 一致，无特殊样式
- 其他 todo 自动让出空间，平滑动画

### 同组内松手

- 更新 `sortOrder`，插入到占位符位置
- 批量重算该组内所有 todo 的 `sortOrder`

### 跨组松手

- `importance` 改为目标组的值
- 插入到占位符位置，重算目标组的 `sortOrder`
- 原组的 `sortOrder` 也重算

### 输入框区域松手

- 拖拽状态下，输入框变为拖拽落点区域
- 视觉：背景变为浅色高亮，显示提示文字"高优先级"
- 拖拽指针进入时高亮加强（边框变色）
- 松手后：`importance` = `max(除被拖拽 todo 外的未完成 todo 的 importance) + 1`，`sortOrder` 为该组末尾
- 除被拖拽 todo 外没有其他未完成 todo 时 importance 保持不变

### 无效区域松手

- 回到原位，无任何变更

### 限制

- 已完成 todo 不可拖拽
- 未完成 todo 不可拖入已完成组

## 技术方案

- 使用 `@dnd-kit` 实现
- 每个重要性组是一个独立的 `SortableContext`
- 输入框区域作为一个独立的 `Droppable`
- 顶层 `DndContext` 管理全局拖拽状态

## 验收标准

- [ ] 长按 500ms 进入拖拽，有缩放反馈
- [ ] 拖拽时 todo 跟随指针，原位留空
- [ ] 同组内松手改变排序，`sortOrder` 正确更新
- [ ] 跨组松手改变 importance 和 sortOrder，两组均正确更新
- [ ] 输入框区域在拖拽时变为落点，松手设为最高优先级
- [ ] 无效区域松手回到原位
- [ ] 已完成 todo 不可拖拽
- [ ] 刷新页面后排序保持
- [ ] 旧 localStorage 数据正常加载
